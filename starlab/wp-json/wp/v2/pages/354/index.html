Background<\/strong><\/h4>\n

To analyze and design an SoS, model-based approaches are generally used as SoS engineering activities. For the model-based engineering, both input and intermediate models should be systematically generated. <SIMVA-SoS Modeler> is a modeling tool for conceptually designing SoS and its component objects such as organizations, CSs, infrastructure, environment, and so on. This tool is developed based on ADOxx metamodeling platform, which supports modeling language development by providing a meta-modeling framework. Based on M2SoS (Meta-model for SoS, See M2SoS page), we developed several model types to represent a target SoS and its components.<\/p>\n

In <SIMVA-SoS Modeler>, 6 major model types are included as follows (Find more information in Modeling Language Development page).<\/p>\n

    \n
  • Integration Model<\/strong>: This model integrates component entities of an SoS, and this model is basically constructed by referencing model instances of other model types.<\/li>\n
  • Goal Decomposition Model<\/strong>: This model defines requirements, tasks, services based on SoS-level common goals.<\/li>\n
  • Organization Model<\/strong>: This model defines an organization and its components (i.e., constituent systems), and this model also includes organization-level components such as contracts, roles, goals, tasks.<\/li>\n
  • CS Capability Model<\/strong>: This model represents a single CS and its features to be used for SoS integration. Thus, CS-level capability is the most important class of CS Capability Model, and CS-level resource, knowledge are also included.<\/li>\n
  • SoS Environment Model<\/strong>: This model represents an SoS-level environment comprising of several environmental elements.<\/li>\n
  • SoS Infrastructure Model<\/strong>: This model represents infrastructure services and resources to be used or accessed by SoS components.<\/li>\n<\/ul>\n

    Since this tool is developed based on ADOxx meta-modeling platform, the ADOxx modeling toolkit should be installed to use a library of <SIMVA-SoS Modeler>. Detailed installation and user guide are included in next sections.<\/p>\n

                \n

              • Java code generator transforms input models into simulatable Java code using predefined classes in SIMVA-SoS.<\/li>\n<\/ul>\n<\/li>\n
              • SIMVA-SoS displays information of imported input files.<\/li>\n
              • SIMVA-SoS supports re-importation of input files.<\/li>\n
              • SIMVA-SoS shows runtime status of model objects during simulation.<\/li>\n<\/ul>\n<\/li>\n<\/ul>\n
